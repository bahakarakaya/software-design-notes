# Software Design Notes

Personal notes on software design principles, with practical coffee shop ☕ themed Python examples.

## SOLID Principles

SOLID is a set of five object-oriented design principles aimed at making software more maintainable, flexible, and scalable.

| # | Principle | Short Definition |
|---|-----------|-----------------|
| S | [Single Responsibility](SOLID/01-SRP.md) | A class should have only one reason to change |
| O | [Open/Closed](SOLID/02-OCP.md) | Open for extension, closed for modification |
| L | [Liskov Substitution](SOLID/03-LSP.md) | Subtypes must be substitutable for their base types |
| I | [Interface Segregation](SOLID/04-ISP.md) | Prefer many specific interfaces over one general-purpose interface |
| D | [Dependency Inversion](SOLID/05-DIP.md) | Depend on abstractions, not on concrete implementations |
